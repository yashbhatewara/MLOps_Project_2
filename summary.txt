# Abu Dhabi Property Rent Prediction Model - Technical Summary

## 🎯 Project Overview
**Objective:** Predict monthly rental prices for properties in Abu Dhabi using machine learning  
**Approach:** Dual modeling - Regression (exact price) + Classification (price category)  
**Best Performance:** R² = 0.90, RMSE = 2,090 AED, MAE = 1,410 AED  
**Key Achievement:** 90% variance explained with ±1,410 AED average prediction error

---

## 📊 Dataset Characteristics
- **Size:** 69,059 properties after cleaning
- **Features:** 21 total (14 original + 3 engineered + target variables)
- **Target Variable:** Monthly rent in AED (normalized from yearly/weekly/daily frequencies)
- **Geographic Scope:** Abu Dhabi properties with location, coordinates, and property attributes

### Key Features:
- **Categorical:** Location, City, Type, Furnishing, Frequency
- **Numerical:** Beds, Baths, Area (sqft), Latitude, Longitude, Age of listing
- **Engineered:** Sqft_per_bed, Total_Rooms, is_high_rise

---

## 🔧 Data Preprocessing Pipeline

### 1. **Data Cleaning**
```
Original → Normalized rent frequencies → Outlier removal → Final dataset
73,000+   →   69,059 properties
```
- Removed top 5% rental outliers (above 95th percentile)
- Filtered properties >10,000 sqft (luxury/commercial outliers)
- Handled missing values in Location, Latitude, Longitude

### 2. **Feature Engineering** ⭐
Created domain-specific features to capture rental dynamics:

| Feature | Formula | Purpose |
|---------|---------|---------|
| **Sqft_per_bed** | Area / (Beds + 1) | Measures living space density |
| **Total_Rooms** | Beds + Baths | Total property capacity |
| **is_high_rise** | 1 if "Apartment" else 0 | Property type indicator |

**Why these matter:** They capture non-linear relationships. A 3-bed with 2,000 sqft feels different than a 3-bed with 1,000 sqft.

### 3. **Target Transformation** 🔥
**Problem:** Rent prices are heavily right-skewed (most properties are affordable, few are luxury)  
**Solution:** Log transformation (`np.log1p()`)
```
Before: Prices range 2,000 → 300,000 AED (skewed distribution)
After: Log-scale normalizes distribution for better model learning
Final: Predictions reversed using np.expm1() for interpretable AED values
```

**Impact:** R² jumped from 0.29 → 0.90 with this single change!

### 4. **Categorical Encoding: Target Encoding**
**Why not One-Hot Encoding?** Location has 500+ unique values → would create 500+ dummy columns

**Target Encoding Strategy:**
- Calculate average rent for each location in training data
- Replace location names with their corresponding averages
- For unseen locations in test data, use global mean as fallback

**Example:**
```
Al Reem Island → avg rent 18,500 AED → encoded as 18,500
Khalifa City → avg rent 12,300 AED → encoded as 12,300
```

This captures location value in a single numerical column!

## 🤖 Modeling Approach

### Algorithm Selection: XGBoost Regressor
**Why XGBoost?**
- Handles non-linear relationships (rent isn't linear with size)
- Built-in regularization prevents overfitting
- Fast training with parallel processing
- Excellent for tabular data (beats Neural Networks on structured data)

### Hyperparameter Optimization
**Method:** RandomizedSearchCV with 5-Fold Cross-Validation  
**Search Space:** 10 random combinations from parameter grid  
**Total Fits:** 50 (10 combinations × 5 folds)

**Parameters Tuned:**
```python
n_estimators: [200, 500, 800]      # Number of trees
learning_rate: [0.01, 0.05, 0.1]   # Step size
max_depth: [4, 6, 10]              # Tree complexity
subsample: [0.7, 0.8, 0.9]         # Row sampling
colsample_bytree: [0.7, 0.8, 0.9]  # Column sampling
gamma: [0, 1, 5]                   # Regularization strength
```

**Best Configuration Found:**
```python
{
    'n_estimators': 200,
    'learning_rate': 0.1,
    'max_depth': 6,
    'subsample': 0.8,
    'colsample_bytree': 0.8,
    'gamma': 0
}
```

## 🎓 Key Learnings & Decisions

### Why Log Transform?
"Rent prices follow a power law distribution—most properties are affordable, few are luxury. Linear models struggle with this. Log transformation normalizes the distribution, allowing the model to learn patterns across all price ranges equally."

### Why XGBoost over Linear Regression?
"Rent isn't linear. A 2-bed in Downtown Abu Dhabi costs more than a 3-bed in a suburb. XGBoost captures these non-linear interactions through decision trees, which linear models can't."

### Why Target Encoding over One-Hot?
"With 500+ unique locations, one-hot encoding would create 500+ columns (curse of dimensionality). Target encoding compresses this into one column while preserving location's predictive power."

### Why 5-Fold Cross-Validation?
"A single train-test split could get lucky/unlucky. Cross-validation tests the model on 5 different splits, ensuring performance isn't a fluke. CV R² of 0.8984 confirms robustness."
